    博客搭建进阶

    # 0. 引言

    经过前面两部分的说明，我们已经熟悉了如何开启一个vuepress或以vdoing为主题的vuepress项目，不过要使博客完整或丰富起来，还需要更深入学习vuepress和vdoing的详细用法(详情可见vuepress官网 (opens new window)和vdoing官网 (opens new window))，由于本文不可能面面俱到，所以内容上主要是对我们学习过程中难懂的点或者一些需要明确的概念浅做解释，主要有目录结构 页面属性与分类 部署

    # 1. 目录结构

    vdoing提供的目录结构如下






     
     
     
     
     
     
     








     
    .
├── .github   (可选，GitHub 相关文件)
│   ├── workflows
│   │   ├── baiduPush.yml (可选，百度定时自动推送)
│   │   └── ci.yml (可选，自动部署)
├── docs (必须，不要修改文件夹名称)
│   ├── .vuepress （同官方，查看：https://vuepress.vuejs.org/zh/guide/directory-structure.html#目录结构）
│   ├── @pages (可选，自动生成的文件夹，存放分类页、标签页、归档页)
│   ├── _posts (可选，专门存放碎片化博客文章的文件夹)
│   ├── <结构化目录>
│   └── index.md (首页)
├── vdoing (可选，本地的vdoing主题)
├── utils  (可选，vdoing主题使用的node工具)
│   ├── modules
│   ├── config.yml (可选，批量操作front matter配置)
│   ├── editFrontmatter.js (可选，批量操作front matter工具)
├── baiduPush.sh (可选，百度推送命令脚本)
├── deploy.sh (可选，部署命令脚本)
│
└── package.json

    高亮部分是前期博客搭建的重点

    docs/@pages 是在dev过程中，自动生成的文件夹，不需要自己配置

    docs/_posts 是碎片文章的存储文件夹，注意其不会在结构化侧边栏和目录页(后续说明)中生成

    docs/<结构化目录> 是博客文章的存储部分，不过其要遵循命名约定，具体说明见《构建结构化站点的核心配置和约定》 (opens new window)

    docs/.vuepress/config.js 和package.json 是配置上的重点，不要忘记dev过程中必要的配置

    ::: tips

    目录结构注释中提到的部署 和涉及不同页面的页面属性 将在之后说明

    :::

    # 2. 页面属性与分类

    在vdoing中，页面的属性是用front matter 说明的，其位于每篇markdown文件的最初部分，其配置详情可见官网说明 (opens new window)，不过官方说明比较详细，我们将按照页面类型进行分类，分别说明不同类页面下front matter配置需要注意的部分。

    # 2.1 文章页

    对于普通的文章页 ，其front matter框架大体如下

    ::: tips

    vdoing具有自动生成front matter的功能，而生成的规则我们在下列框架中用注释进行了表明。

    :::




     












    ---
title: 标题					#默认：文件名
date: 2020-02-21 14:40:19	   #默认：文件修改时间
permalink: /pages/a1bc87	   #该部分后续说明
categories:					  #由结构化目录生成
  - 分类1
  - 分类2
tags:						  #自动生成，但内容为空
  - 标签1
author:
 name: 作者
 link: https://xxx.com
sticky: 1					  #置顶选项，在推送列表中置顶，数值越低，排名越高
---

    permalink 页面的永久链接，其默认值为pages/ + 6位字母与数字的随机码 ，其在意义上可以理解为该页面的一个永久地址，不随路径而改变，因而，当md文件所处的文件夹改变时，通过永久链接仍能访问对应文件，这样的设置提高了修改的灵活性，同时对于重要的文章建议自己编辑永久链接，方便之后改动时检索。

    # 2.2 目录页

    目录页的目录指的不是某篇文章中左右侧的目录，而是单击导航栏的所出现的页面，如下是vdoing作者博客中的一个目录页

    其front matter结构如下

     
     
     
     
     
     




     



    pageComponent:				   #后续说明
  name: Catalogue
  data:
    path: 01.前端               #目录页对应内容所在的路径
    imgUrl: /img/web.png       #下图对应的图片
    description: description   #下图中description文本对应的即该配置的描述
title: title				   #下图中title对应的文本即该配置的描述
date: 2020-03-11 21:50:53
permalink: /web/
sidebar: false                 
article: false                 #后续说明
comment: false
editLink: false

    想要让一个页面为目录页则必须有上述代码中高亮部分

    一是在pageComponet下声明用了目录组件Catalogue

    二是需要将article一配置的值设为flase，说明这不是一个普通的文章页

    ::: warning:

    注意区分目录页front matter中的path和permalink两配置，前者是目录页对应内容的地址，后者是目录页本身的地址。

    :::

    # 3.3 首页

    首页即docs/index.md或docs/README.md文件所对应的页面，不过要生成vdoing所对应的格式化的首页，还需要在front matter最开始声明home: true才可以，其基本结构如下

    home: true
# heroImage: /img/web.png
heroText: Evan's blog
tagline: Web前端技术博客，积跬步以至千里，致敬每个爱学习的你。
# actionText: 立刻进入 →
# actionLink: /web/
# bannerBg: auto # auto => 网格纹背景(有bodyBgImg时无背景)，默认 | none => 无 | '大图地址' | background: 自定义背景样式       提示：如发现文本颜色不适应你的背景时可以到palette.styl修改$bannerTextColor变量

features: # 可选的
  - title: 前端
    details: JavaScript、ES6、Vue框架等前端技术
    link: /web/ # 可选
    imgUrl: /img/web.png # 可选
  - title: 页面
    details: html(5)/css(3)，前端页面相关技术
    link: /ui/
    imgUrl: /img/ui.png
  - title: 技术
    details: 技术文档、教程、技巧、总结等文章
    link: /technology/
    imgUrl: /img/other.png

# 文章列表显示方式: detailed 默认，显示详细版文章列表（包括作者、分类、标签、摘要、分页等）| simple => 显示简约版文章列表（仅标题和日期）| none 不显示文章列表
# postList: detailed
# simplePostListLength: 10 # 简约版文章列表显示的文章数量，默认10。（仅在postList设置为simple时生效）
# hideRightBar: true # 是否隐藏右侧边栏

    由于示例front matter中注释十分详细，所以不再多做解释

    # 3. 部署

    部署可理解为当你将博客内容完成后，发布到线上的过程，本文将只讲述发布到利用Github Pages服务部署到Github上的过程

    其具体步骤如下

    1、编译打包

    cd 项目目录名
npm run build

    打包完的项目会生成在docs/.vuepress/dist中

    2、设置github账号配置信息

    git config --global user.name "xintianyucn"
git config --global user.email "iworkinsky@gmail.com"

    测试配置信息是否正确 ，如果正确，执行下面命令后会有success信息

    ssh -T git@github.com

    3、进入打包后的目录

    cd 项目目录名/docs/.vuepress/dist

    初始化本地git环境

    git init

    把当前目录所有文件加入到本地git库

    git add -A
git commit -m "deploy"

    设置对应远程仓库映射

    git remote add origin  git@github.com:发布地址

    强制把本地库上传覆盖远程仓库

    **master:gh-pages ** 是能发布的必须的分支名称

    git push -f git@github.com:地址 master:gh-pages

    4、到浏览器查看效果

    到浏览器输入你上传的地址查看效果，注意一般上传后过1、2分钟才能看到更新后的情况

    注意

    这只是基于Github Pages部署的基本步骤，因为官方文档是基于使用者拥有一定git基础，在基本步骤上不太详尽，故我们将其详细列出，不过要实现更丰富的功能，实现部署的升级，可以查看官方的部署说明 (opens new window)，其中包括了脚本部署等更适应化的方式，同时为了实现访问速度的提升，也可选用国内Gittee平台等服务，本文在此便不再详述
